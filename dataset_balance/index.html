B. 다양성 상한 — 어디가 병목인가

MCP 계열의 76k는 내재적 상한(문제 아님). 도구 카탈로그가 125개로 고정이고, 시나리오 76,251개는 공식 데이터셋(SGD/BFCL 파생)의 커버리지 그대로다. 벤치마크 정합성을 위해 의도적으로 공식 시나리오에 묶인 것 — 무한 확장 대상이 아니라 커버리지를 다 쓰는 게 목표.

Memory · Persona는 우리가 자유롭게 키우는 kind. 공식 데이터셋에 묶이지 않은 자체 설계라 오늘 Context DB 다양성 소스를 대폭 확장(§C-3, §C-4). 결과: Memory/Persona는 무한에 가까워 지금 비율(각 11.9%)보다 더 큰 몫으로 안전하게 확장 가능.

C. 출처 인용 & 방법론

C-1. MCP (도구/서비스) — 76,251 시나리오

Reference. Google Schema-Guided Dialogue (SGD) — Rastogi et al., Towards Scalable Multi-Domain Conversational Agents, AAAI 2020 (arXiv:1909.05855). 도구 택소노미는 BFCL(Berkeley Function-Calling Leaderboard, Gorilla)에서 파생. 커버리지 목표는 MCPToolBench++급 125 tool · 24 domain.

파이프라인. build_mcp_taxonomy.py(BFCL/SGD)에서 도구 스키마를 뽑아 build_mcp_pool.py가 발화 가능·TTS-safe 시나리오 76,251개를 캐시(mcp_scenario_pool.json). 각 대화는 시나리오 1개를 서수(무재사용)로 소비 → ask → <ret> → CALL(JSON) → SPAN (tool result)… → grounded 답변.

C-2. RAG (지식 그라운딩 + abstain) — ≈429,431행

References. Natural Questions — Kwiatkowski et al., TACL 2019 (~91,535 train+val) · HotpotQA — Yang et al., EMNLP 2018 (arXiv:1809.09600; distractor+fullwiki ~90,564) · SQuAD 2.0 — Rajpurkar et al., ACL 2018 (arXiv:1806.03822): answerable ~92,749 → rag_single, unanswerable ~49,443 → abstain.

파이프라인. 각 QA 행을 서수(무재사용)로 소비. answerable → gold context/answer를 REF로 실어 grounded 답변, HotpotQA는 supporting_facts를 gold ref로. unanswerable(abstain) → 단일턴, reference=NOINFO_REF로 "모름"을 정직하게 회피(환각 금지). 즉 abstain은 별도 셋이 아니라 SQuAD2의 답-없는 질문 그대로.

C-3. ⭐ Memory Talk — 자체 설계 (외부 데이터셋 없음)

방법론. 대화별 User Context DB를 생성: profile(이름·도시·국적·나이) + saved_memories(날짜 스탬프된 정밀 사실) + conversation_summaries(과거 대화 요약). 유저가 저장기억을 물으면 백엔드가 해당 라인을 <ret>로 회수, 모델은 그 REF만 읽어 답한다(그 밖 사실 날조 금지).
다양성(오늘 확장). 카테고리 골격 9 → 36개(집·직장·통근·가족·건강·복약·기기·차·구독·저축·학습·미디어·기념일·비상연락 …), 값 풀 평균 6→11 → 카테고리-세트 조합 336 → 1.35×10⁸. 과거대화 요약도 6→28종이며 이름·숫자·날짜를 담아 "이전 대화의 매우 상세한 것"까지 정밀 회수 가능. 유저당 표본 메모리 5~9 / 요약 2~4.
대응 개념. MoshiRAG-style memory-augmented turn을 우리 Context DB로 구현(공식 학습셋 아님 → 자체 생성).

C-4. ⭐ Persona Talk — 자체 설계 (외부 데이터셋 없음)

방법론. 에이전트 페르소나만으로 <ret> 없이 응답(상담사 톤 유지). 페르소나 카드 = trait × role × focus 조합 + 음절 생성 에이전트 이름 + 대화별 유저 프로필.
다양성(오늘 확장). 성격 골격 800 → 10,080(trait 24 × role 14 × focus 30) × 생성 이름(무한) × 풍부해진 프로필. role은 상담/코칭/멘토 계열로 한정해 톤 일관성 유지.
대응 개념. PersonaPlex-style 페르소나 부여를 자체 생성기로 구현(공식 학습셋 아님).

C-5. 기타 자체 방법론

(a) MCP 파생 변형 — deferred(결과가 몇 턴 뒤 도착, MOSHI-DEFERRED 라벨로 span을 발표 0.3–0.8 s 전 주입), falsestart(도구 호출 전 발화 중단·재시작), noinfo(도구가 빈 결과 → 정직한 miss)는 SGD 시나리오 위에 우리 고유 full-duplex 행동을 얹은 것. 시나리오 출처는 C-1과 동일, 행동 라벨은 자체 설계.
(b) Safety / Decline — 외부 refusal 벤치마크 + 자체 카테고리 taxonomy의 결합(load_safety_pool, 카테고리 균형 ~2,860 slots).
References (외부, 다운로드). Do-Not-Answer(LibrAI, 939 · 위해 유형별) · BeaverTails(PKU-Alignment, unsafe 1,483 · 14 harm categories) · AdvBench(Zou et al. 2023, 520 harmful behaviors) · XSTest In-Character Refusals(Röttger et al. 2024, unsafe ~2,880) → 일반 위해(폭력·범죄·프라이버시침해·성인 등) 커버.
자체 카테고리(외부셋이 안 다루는 것). 물리적 위험(30) · 정파 정치(12) · 지정학/주권 = 중화사상(대만·티베트·홍콩·센카쿠·남중국해·독도 등 12) · 프라이버시(8) · 자해/위기(3) · 전문조언 의료/법률/금융(8). 각 카테고리 ~7%씩, 외부 위해 ~58%.
Handling(카테고리별). 위험/불법 → 명확히 거절 + 위험 설명 · 정치/지정학 → 엄격 중립, 편들지 않기(정중히 회피) · 프라이버시 → 접근·공유 거부 · 자해 → 공감 + 실제 지원 안내 · 전문조언 → 일반 정보 + 전문가 권유. 압박에도 입장 유지, <ret> 없음(판단).
(c) to_other — 자체 라우팅 방법론: 요청이 제3의 에이전트/사람에게 넘겨져야 할 때 to_other 라벨로 위임을 표현. MCP/RAG 시나리오에서 파생.
(d) fake_span — 자체 spurious-retrieval robustness: <ret>가 부적절한 시점/무관한 내용으로 발화되도록 주입해, 모델이 관련 없는 span을 무시하고 문맥을 유지하는지 평가. MCP/RAG 시나리오에서 파생.
(e) Paralinguistic (pause·background·hesitation) — kind가 아니라 교차주입 행동: ONEPASS_SYS가 전 대화에 통계적으로 무음/휴지·배경소음·머뭇거림을 섞음. 별도 데이터셋 없이 생성 규칙으로 표현.

현재 양산: EN-only 50k (풀의 ~3.6% 표본). BASE 비율 유지 · 시드 무재사용. 표는 gen30 BASE(42 EN 슬롯) 실측 비율 기준. 행수는 mass.py 데이터 로더 기준.
