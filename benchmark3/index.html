Full-Duplex-Bench v3

(2026) · arXiv:2604.04847 — 실제 사용자의 disfluency(더듬거림) 속에서도 모델이 견고하게 올바른 tool을 호출하는가를 측정하는 벤치마크. 핵심 질문은 “사용자 발화가 filler·pause·hesitation·false start·self-correction으로 지저분해져도, 모델은 흔들리지 않고 정확한 tool orchestration을 유지하는가”다. 이 문서는 단독으로 읽을 수 있도록 동기·지표 수식·자극·데이터·한계를 모두 담는다.

음원 안내. 관련 카드에는 우리가 생성한 대화 음원이 붙어 있다. 모두 스테레오이며 왼쪽 채널 = moshi, 오른쪽 채널 = 사용자다. 플레이어는 해당 현상이 나타나는 지점부터 재생된다.

0왜 disfluency + tool인가

기존 tool-use 벤치마크는 깨끗하게 정제된 지시문을 전제한다 — “샌프란시스코행 항공편을 찾아줘” 같은 완결된 명령. 그러나 실제 음성 사용자는 그렇게 말하지 않는다. 실사용자는 더듬는다: “음… 샌프란— 아니, 시카고로 가는 항공편… 잠깐, 편도로.” 발화는 filler로 시작하고, 문장 중간에 멈추며(pause), 같은 말을 반복하고(hesitation), 의도를 철회했다가(false start), 파라미터를 도중에 갈아치운다(self-correction).

진짜 난제는 그 잡음 속에서 정확한 tool orchestration을 유지하는 것이다. 모델은 무효가 된 문맥을 버리되 없는 tool을 지어내선 안 되고(hallucinated tool call 금지), 갱신된 파라미터로 상태를 되감아야(state rollback) 하며, 사용자가 멈칫하는 동안 성급히 잘못된 호출을 던져선 안 된다.

설계의 핵심 비대칭. disfluency는 오직 사용자 쪽에서만 발생한다 — 사용자는 더듬어도 된다 (ROBUST 대상) 모델은 매끄러워야 한다 (FLUENT 요구) 모델이 스스로 내용 없는 filler(“음…”, “어…”)를 내면 PENALTY다. filler는 자극(사용자)의 속성이지 응답(모델)의 속성이 아니다. 이 비대칭 — 입력은 지저분해도 되지만 출력은 깨끗해야 한다 — 이 v3 평가의 뼈대다.

1지표 — Tool Selection F1 · Arg Accuracy · Pass@1

tool orchestration은 chained이다: 한 사용자 요구가 여러 tool의 연쇄 호출로 이어질 수 있다. 각 평가 사례마다 기대되는 tool 집합 \(T^\*\)와 모델이 실제로 부른 tool 집합 \(\hat T\)를 비교한다. Tool Selection F1은 정밀도·재현율의 조화평균:

$$ \mathrm{P}=\frac{|\hat T\cap T^\*|}{|\hat T|},\qquad \mathrm{R}=\frac{|\hat T\cap T^\*|}{|T^\*|},\qquad \mathrm{F1}=\frac{2\,\mathrm{P}\,\mathrm{R}}{\mathrm{P}+\mathrm{R}} $$

tool을 맞게 골랐더라도 인자(argument)가 틀리면 소용없다. Arg Accuracy는 각 호출의 인자 묶음을 기대값과 대조해 GPT-4o가 채점한다(값 정합·타입·정규화 반영):

$$ \mathrm{ArgAcc}=\frac{1}{|C|}\sum_{c\in C}\mathbb{1}\!\left[\text{GPT-4o가 호출 }c\text{의 인자를 기대값과 일치로 판정}\right] $$

가장 엄격한 통합 지표는 Pass@1이다. 단 한 번의 시도에서 기대한 tool들을 정확히 그 집합대로 부르고 모든 인자가 완벽해야만 통과한다 — 부분 점수 없음:

$$ \mathrm{Pass@1}=\mathbb{1}\!\left[\hat T=T^\* \;\wedge\; \forall c\in C:\ \mathrm{args}(c)=\mathrm{args}^\*(c)\right] $$

여기에 Response Quality(GPT-4o \([0,5]\), 사용자에게 돌려주는 자연어 응답의 관련성·일관성·적응성)를 병행한다.

대화 타이밍 보조 지표

tool 정확도와 별개로 full-duplex 상호작용의 타이밍도 측정한다. turn-take rate(응답 기회에서 실제 발언권을 잡은 비율), interruption(모델 발화 onset이 사용자 발화 offset보다 이른 경우 — 성급한 끼어듦), 그리고 세 종류의 latency:

$$ \mathrm{Latency}_{\text{first}}=t^{\text{model}}_{\text{on}}-t^{\text{user}}_{\text{off}},\quad \mathrm{Latency}_{\text{tool}}=t^{\text{tool-call}}-t^{\text{user}}_{\text{off}},\quad \mathrm{Latency}_{\text{task}}=t^{\text{task-done}}-t^{\text{user}}_{\text{off}} $$

first는 첫 자연어 반응까지, tool은 첫 tool 호출 발화까지, task는 연쇄 전체가 완료돼 최종 응답이 나올 때까지의 지연이다.

Filler rate — 모델 filler는 감점

사용자 쪽 filler는 자극이지만, 모델이 내용 없는 filler를 산출하면 PENALTY다. Filler rate는 모델 발화 중 의미 없는 채움말(“um”, “uh”, “음…”, 무내용 지연발화)이 차지하는 비율로, 낮을수록 좋다:

$$ \mathrm{FillerRate}=\frac{\#\{\text{모델의 무내용 filler 토큰}\}}{\#\{\text{모델 발화 토큰}\}}\ \downarrow $$
비대칭 재확인. 같은 “um/uh”라도 사용자 채널에 있으면 robustness를 시험하는 자극이고, 모델 채널에 있으면 fluency를 깨는 감점 요인이다. 모델은 사용자가 더듬는 만큼 더 침착하고 매끄럽게 응답해야 한다.

다섯 disfluency는 모두 사용자 발화 쪽의 현상이며, 모델은 각각에 대해 견고(ROBUST)해야 한다.

자극

Chained tools — Pass@1 음원

한 발화가 두 개 이상의 tool 연쇄로 이어질 때, 정확히 그 tool들을 완벽한 인자로 불러야 Pass@1을 통과한다.

tool 결과가 즉시 오지 않고 늦게 도착할 수 있다. 모델은 결과가 돌아오는 순간 대화를 자연스럽게 이어 선제적으로 발표해야 한다.

자극은 네 개의 tool 도메인에 걸쳐 구성되며, 각 도메인은 고유한 tool 세트와 파라미터 공간을 가진다.

3한계 · 평가 대상

① 점수 표는 싣지 않는다. 본 문서는 벤치마크의 설계·자극·지표 정의를 서술한다. 검증되지 않은 모델별 수치를 날조해 표로 제시하지 않는다 — 실제 채점 결과가 확보되기 전까지는 정성적으로만 서술한다.

② Arg Accuracy·Response Quality의 심판이 GPT-4o. 인자 정합·응답 품질 채점을 LLM 심판에 의존하므로, 심판의 편향·일관성이 곧 지표의 상한이다. Tool Selection F1·Pass@1은 집합/문자열 대조라 결정적이지만, 인자 정규화(동의어·표기 변형) 판정에는 여전히 주관이 개입한다.

③ disfluency의 인위성. filler·pause·hesitation·false start·self-correction은 통제된 규칙으로 삽입한 합성 자극이다. 실제 사용자의 더듬거림 분포와 완전히 일치하지 않을 수 있으며, 실사용 로그 기반 검증이 후속 과제다.

④ 모델 fluency 판정의 경계. “모델 filler는 감점”이라는 규칙은 명확하지만, 자연스러운 담화 표지(“네”, “알겠습니다”)와 무내용 filler(“음…”)의 경계가 항상 또렷하진 않다. Filler rate 집계 시 이 경계 정의가 결과를 좌우한다.

요약. v3의 기여는 “깨끗한 지시문” 가정을 깨고 실사용자 disfluency 하의 tool orchestration robustness를 분리 측정한다는 점이다 — 사용자는 더듬어도 되고, 모델은 매끄러워야 한다는 비대칭을 지표(Filler rate penalty)와 자극 설계 양쪽에 심었다.
