① 50k EN 학습데이터 (텍스트)완료

영문 대화 49,956개 / 1,200 parts 생성. Dueta와 동일 폴더/스키마로 저장(/autodl-fs/data/ksh/duetaspan_v1/) + span·retrieval·voice 확장 키를 복구 가능하게 추가. 시드 무재사용, 길이 축소 없음, 품질 규칙서(ONEPASS_SYS) 무손실.

② 처리량 진단완료

실측 ~5,000/h. 원인 = prefill-bound: 프롬프트 ~5,500토큰(규칙서 3,880 포함)을 매 요청 재-prefill, 출력은 ~350토큰 → 토큰작업의 94%가 프롬프트. 품질 유지가 우선이라 규칙서는 유지. MCP 다양성 상한은 공식 SGD 76k 시나리오(정상).

③ Memory · Persona 다양성 대폭 확장완료

Memory: 카테고리 골격 9 → 36개(집·직장·통근·가족·건강·복약·기기·차·구독·저축·학습·미디어·기념일 …), 값 풀 6→11 → 조합 336 → 1.35×10⁸. 과거대화 요약 6→28종에 이름·숫자·날짜를 담아 "이전 대화의 매우 상세한 것"까지 정밀 회수. 2,000유저 충돌 0.
Persona: 성격 골격(trait×role×focus) 800 → 10,080 × 무한 생성 이름 × 풍부해진 프로필. (소스 커밋 72ff0b5)
