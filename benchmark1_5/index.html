Full-Duplex-Bench v1.5

(2025) · arXiv:2507.23159 — full-duplex 음성대화 모델이 두 화자의 음성이 실제로 겹치는 구간(overlap)에서 발언권을 양보할지·무시하고 이어갈지를 옳게 고르는지를 자동 지표로 평가하는 벤치마크. v1이 순차적 turn-taking을 봤다면, v1.5는 “겹쳐 말하기”가 일어난 바로 그 순간의 모델 행동을 본다. 이 문서는 단독으로 읽을 수 있도록 동기·행동 분류·지표 수식·네 시나리오·우리 데이터·한계를 모두 담는다.

0왜 overlap을 따로 평가하나

v1(arXiv:2503.04721)은 침묵·맞장구·발언권 전환·끼어들기를 순차적 turn-taking 관점에서 다뤘다. 즉 “언제 말을 시작/유지하는가”가 핵심이었다. 그러나 실제 대화에서 가장 다루기 어려운 순간은 두 채널의 음성이 물리적으로 겹치는 구간이다 — 사용자가 모델 발화 위로 끼어들거나, 짧은 맞장구를 얹거나, 옆사람에게 말을 걸거나, 배경에서 다른 화자가 떠드는 상황.

이때 모델이 내려야 하는 결정은 이진이 아니다. 겹침이 들렸으니 무조건 멈춘다면 맞장구·배경음에도 매번 발화가 끊겨 대화가 무너지고, 겹침을 무시하고 계속 말한다면 진짜 끼어들기를 놓친다. 핵심은 겹침의 종류를 구분해 양보(yield)·재개(resume)·무시를 옳게 선택하는가다.

설계의 뼈대. v1.5는 하나의 겹침 사건에 대해 모델 행동을 4범주로 분류하고, 겹침 시작 → 말 멈춤 시각(stop-latency)과 겹침 → 응답 시작 시각(response-latency)을 잰다. 시나리오마다 ‘정답 행동’이 다르다 — 같은 “겹침”이라도 끼어들기엔 YIELD, 맞장구엔 RESUME이 정답이다. 이 조건부성이 v1.5 설계의 핵심.

1지표 — 행동 4범주와 stop / response latency

겹침 사건 하나마다, 모델 채널의 반응을 다음 행동(behavior) 4범주로 분류한다:

$$ \text{behavior}\in\{\;\textbf{Respond},\;\textbf{Resume},\;\textbf{Uncertain},\;\textbf{Unknown}\;\} $$
Respond
겹침을 새 발언권 요구로 받아들여 말을 멈추고(yield) 사용자의 새 발화에 응답. 끼어들기(interruption)의 정답.
Resume
겹침에도 흔들리지 않고 원래 발화를 이어감. 맞장구·제3자 발화·배경음의 정답 — 멈추지도, 새로 수리(repair)하지도 않는다.
Uncertain
멈췄다가 다시 이어가거나 어정쩡하게 흐려지는 등, Respond와 Resume 어느 쪽으로도 깔끔히 분류되지 않는 애매한 반응.
Unknown
VAD로 유효한 발화 경계를 잡을 수 없어 판정 불가한 경우.

행동 경계와 지연 측정은 Silero VAD로 모델·사용자 채널의 발화 구간(on/off)을 검출해 계산한다. 두 지연을 정의한다:

$$ \text{stop-latency}=t^{\text{model}}_{\text{off}}-t^{\text{overlap}}_{\text{start}},\qquad \text{response-latency}=t^{\text{model}}_{\text{on(new)}}-t^{\text{overlap}}_{\text{start}} $$

stop-latency = 겹침이 시작된 시점에서 모델이 실제로 발화를 멈춘 시점까지의 시간(양보 반응성). YIELD가 정답인 시나리오에서 작을수록 좋다. response-latency = 겹침 시작에서 모델이 새 발화를 개시하기까지의 시간(응답 신속성).

v1이 하나의 TO(Takeover) 위에 목표 방향만 뒤집었다면, v1.5는 하나의 겹침 사건 위에서 정답 행동(Respond vs Resume)을 시나리오마다 바꾼다. 지표는 “겹침을 옳게 해석했는가(행동 분류)”와 “얼마나 반응이 좋은가(stop / response latency)”의 두 축이다.

2네 시나리오 — 자극 · 정답 행동 · 우리 데이터 · 음원

음원 안내. 아래 시나리오 카드의 음원은 모두 스테레오이며 왼쪽 채널 = moshi, 오른쪽 채널 = 사용자다. 플레이어는 겹침 현상이 나타나는 지점부터 재생된다. 각 시나리오는 “같은 겹침, 다른 정답 행동”을 대비해 듣는 것이 핵심이다 — 특히 ①(YIELD)과 ②(RESUME)의 차이에 주목.
① User Interruption — 진짜 끼어들기에 발언권을 양보하는가
자극 구성
모델이 한창 발화하는 중, 약 4초 gap 뒤에 사용자가 모델 발화 위로 겹쳐 끼어든다. 새로운 요구/질문을 담은 실질 발화다.
정답 행동 · 지표
Respond — 모델은 YIELD하고 새 요구에 맞춰 repair. stop-latency↓(빨리 멈춤) · response-latency↓.
즉시 양보하고 새 요구에 맞는 응답으로 수리. ✗ 무시하고 계속 말함 = missed interruption.
우리 데이터 — 사용자만 moshi를 끊는 단방향, moshi 턴을 하이픈으로 절단하고 사용자가 겹쳐 든 뒤 적응:
[moshi ] «끊김» 그가 말하길, 그 느낌이 너무 강렬해서— [사용자] «끼어듦» 잠깐만요. 크리켓이라고요? 정말요? [moshi ] (불평 없이 새 요구에 즉시 적응 → 근거 기반 응답)

② User Backchannel — 짧은 맞장구에는 흔들리지 않고 이어가는가
자극 구성
모델 발화 중 사용자가 yeah / uh-huh / mm-hmm / totally 같은 짧은 맞장구(backchannel)를 겹쳐 얹는다. 발언권을 가져가려는 신호가 아니라 “듣고 있다”는 청자 신호다.
정답 행동 · 지표
Resume — 멈추지 말고 발화를 이어감, 수리하지 않음. ①과의 차이를 구분하는 것이 핵심.
맞장구를 무시하고 원래 발화를 매끄럽게 이어감. ✗ 맞장구에 멈추거나 새로 응답 = false yield.
우리 데이터 — moshi 발화의 유성 구간 위에 사용자의 짧은 맞장구를 겹치고, moshi는 발화를 끊지 않고 이어감:
[moshi ] Nassau has some excellent snorkeling locations. The coral reefs just off… [사용자] «맞장구, 상대 발화 위 겹침» Yeah. [moshi ] (멈춤 없이 발화 계속 → resume)
음원 L=moshi / R=사용자 · ▶ 약 0:21 — moshi가 클로제의 16골 기록을 길게 설명하는 동안 사용자가 “Yeah.” 짧은 맞장구만 얹지만, moshi는 멈추지 않고 발화를 이어간다.
③ Talking to Others — 자신이 수신자가 아님을 알고 침묵/재개하는가
자극 구성
사용자가 모델이 아니라 제3자에게 말한다(예: “Sam—grab the file”). 겹침은 있으나 모델은 수신자가 아니다. 음향적으로 대상 발화는 -8dB, HF shelf(고역 감쇠), 반사 2회로 “옆으로 향한 말”처럼 처리.
정답 행동 · 지표
Resume — 자신에게 온 말이 아니므로 침묵/재개. 제3자 발화에 반응해 발언권을 낚아채면 실패.
수신자가 아님을 인지하고 원래 흐름 유지. ✗ 옆사람 말에 응답 = 오귀속(mis-addressing).
우리 데이터 — 사용자가 잠깐 딴 사람에게 말한 뒤 대화로 복귀, moshi는 그 사이 침묵/재개:
[사용자] «제3자에게, -8dB·반사» Sorry, hold on— yes, two sugars please. [사용자] «복귀» Okay, back… Where were we? [moshi ] (제3자 발화에 반응하지 않고 원래 맥락 유지 → resume)
음원 L=moshi / R=사용자 · ▶ 약 1:04 — 사용자가 잠깐 딴 사람에게 “Hey, did you remember to take the trash out?”라고 말한 뒤 “Sorry about that. Back to cats.”로 대화에 복귀하고, moshi는 그 사이 반응하지 않고 원래 맥락을 유지한다.
④ Background Speech — 무관한 배경 음성을 무시하고 이어가는가
자극 구성
대화와 무관한 다른 화자의 배경 음성이 반향(reverberant)을 띤 채 겹쳐 든다. 사용자가 모델에게 하는 말이 아니라 환경 소음에 가까운 발화다.
정답 행동 · 지표
Resume — 배경 화자에 반응하지 말고 원래 발화를 이어감.
배경 음성을 발언권 신호로 오인하지 않고 발화 유지.
우리 데이터
음원 L=moshi / R=사용자 · ▶ 약 1:49 — 대화와 무관한 배경 발화 “Did you remember to take the trash out?”가 겹쳐 들지만 moshi는 반응하지 않고 원래 화제(Marni Nixon)를 이어간다.
정답 행동 요약. ① User Interruption: Respond(YIELD·repair) ② User Backchannel: Resume ③ Talking to Others: Resume(침묵/재개) ④ Background Speech: Resume
겹침은 하나지만 정답 행동은 ①만 YIELD, ②③④는 이어가기. 이 비대칭을 옳게 가르는가가 v1.5의 시험이다.

3평가 대상 · 한계

v1.5는 full-duplex로 동작하는 음성대화 모델을 겹침 상황에 노출시켜 위 행동 4범주와 stop / response latency로 평가한다. 평가 대상으로는 end-to-end 계열(예: Moshi, dGSLM), cascaded 계열(예: Freeze-Omni), 상용 실시간 계열(예: Gemini Live) 등 v1에서 다룬 것과 같은 부류의 full-duplex 모델이 겹침 처리 관점에서 시험된다.

주의 — 점수 표는 싣지 않는다. 정확한 leaderboard 수치를 확정할 수 없으므로 이 문서는 임의의 점수 표를 만들지 않는다. v1.5의 본질은 특정 순위가 아니라 지표 정의(행동 4범주 · stop / response latency), 네 시나리오의 조건부 정답 행동, 그리고 우리가 생성한 데이터(①②③④ 네 시나리오 모두 음원 커버)에 있다.

알려진 한계.
① 겹침 유형의 조건부성 — 같은 물리적 겹침이라도 정답이 YIELD/RESUME으로 갈리므로, 유형 오분류가 곧 행동 실패로 이어진다.
② 음향 합성 — 제3자·배경 발화는 후처리 믹싱(감쇠·HF shelf·반향)으로 합성한 것이라, 실제 원거리 마이크 캡처와는 음향 특성이 다를 수 있다.
③ 영어 한정 자극 — 다국어 겹침 처리 일반화는 미검증. (본 데이터셋은 한국어 생성으로 이 갭을 겨냥.)
