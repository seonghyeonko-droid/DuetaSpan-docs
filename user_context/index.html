🧠 User Context DB — moshi의 개인 메모리

유저별 개인 컨텍스트를 내부지식으로 보유. moshi가 이걸 들고 대화하되, 두 경로로 나뉩니다.

A. 프롬프트 내재 — <ret> 호출 안 함

유저 이름·지역·국적·언어/호칭·성별·나이 등 상시 관련·저정밀 속성 + moshi 자신의 정체성(페르소나). 시스템 프롬프트에 담겨 검색 없이 인삿말·개인화·기본 단위를 자연스럽게.
⚠️ 핵심 학습신호 — over-retrieval 방지: "너의 이름 뭐야?"(moshi 페르소나) · "내 이름 뭐였지?"(유저 프로필) · 인삿말처럼 바로 답 가능한 질문엔 절대 <ret>를 부르지 않는다. 불필요한 retrieval을 안 하는 판단력을 학습.

B. 메모리 RAG — <ret> 필수

정밀·인자결정·에피소드 사실(집/직장 주소·식이·알레르기·반려동물·좌석선호·과거대화). moshi가 검색해야만 정확한 도구 인자를 넘기고("집까지 길안내"→주소 retrieve→내비 인자), "기억나?" 회상에 답함. span 출처 = user_memory(내부), 외부 world-knowledge span과 구별. 검색은 백엔드가 담당.
메모리 = superset: A의 프로필 사실([profile] …)도 여기 함께 기록 → 검색이 모든 걸 커버(retrieval 용이). 단순 질의("이름이 뭐야?")는 페르소나/프롬프트에서 바로 답할 수 있어 <ret> 불필요, 다만 메모리에도 있어 검색으로도 커버.

포맷 근거(리서치): ChatGPT 메모리 = 타임스탬프 단문 saved-fact([2026-05-02] User is vegetarian.) + 날짜별 conversation summary, RAG 검색 주입 · OpenAI Memory · reverse-eng · 페르소나 조건화 = PersonaPlex arXiv:2602.06053. 학습: 프로필 A→시스템 프롬프트 조건화 / 메모리 B→Context Span(R_CTXSPAN 마스킹·검색).

동작 예시 (Stage 2에서 이 메모리로 생성될 대화)
[u] 지금 몇 시야? [m] <ret> … ⟵ 메모리에서 위치·시간대(예: Asia/Seoul, UTC+9) retrieve → 시계 도구에 tz 인자 → "지금 서울은 오후 3시 20분이에요." [u] 저번에 말한 그 레스토랑 기억나? [m] <ret> … ⟵ 메모리 회상(외부지식 아님) → "네, ○○○ 말씀이죠." [u] 집까지 가는 길 안내해줘. [m] <ret> … ⟵ 집 주소 retrieve → 내비 도구 origin/destination 인자 ────── ⬇ 아래는 <ret> 절대 호출 안 함 (over-retrieval 방지) ────── [u] 너의 이름 뭐야? [m] (NO <ret>) moshi 페르소나에서 바로 → "저는 모시예요." [u] 안녕! / 내 이름 뭐였지? [m] (NO <ret>) 프롬프트 프로필에서 바로 → "안녕하세요, ○○님!"
