v1 · turn-taking & overlap  arXiv 2503.04721  · 데이터셋 repo/dataset

Lin et al. — 사람 청취 평가 대신 재현 가능한 자동 지표로 full-duplex 모델의 턴테이킹 4역량을 분리 측정. 아래에 지표 정의(식)·자극 구성·목표 방향·왜 그 지표가 그 과제를 검증하는지를 전부 기술.

공통 원자 지표 — Takeover (TO) / Takeover Rate (TOR) 평가 시점마다 모델 채널을 이진 판정: TO = 0 if (silence or backchannel), 1 otherwise (= floor takeover), TOR = (1/N) Σᵢ TOᵢ (N = evaluated turns). backchannel은 duration < 1 s AND < 2 words인 listener signal로 엄격 정의되어 TO = 0으로 집계. full-duplex의 실패 모드는 (a) false takeover(가져오면 안 될 때 floor를 가져옴)와 (b) missed turn-take(가져와야 할 때 못 가져옴) 둘뿐 — TO는 그 atomic event 자체라서, 과제별로 target direction만 뒤집으면(①② TOR↓=0, ③④ TOR↑=1) 4개 task를 하나의 일관된 척도로 검증한다. 이 대칭이 v1 설계의 핵심.
① Pause Handling — “speaker가 아직 turn을 hold 중임을 아는가” User: "I want to go to … [turn-internal pause 0.4–1.0 s] … Paris." Stimulus 구성: CANDOR corpus(850-hour two-channel real conversations)에서 turn-internal pause 0.4–1.0 s 구간 216 samples + ChatTTS [uv_break]로 합성한 137 samples. 0.4–1.0 s 범위인 이유 = 영어 대화에서 ~1 s가 natural pause의 지각적 상한(perceived upper bound).
Metric · target: TOR ↓ (ideal = 0).
왜 이 metric인가: turn-internal pause는 TRP(Transition Relevance Place)가 아니다 — 여기서 발화를 시작하면 “silence = my turn”이라는 잘못된 heuristic을 쓴다는 직접 증거. backchannel은 TO = 0이라 penalty가 없으므로, 이 metric은 정확히 “silence or backchannel only”라는 task 정의 그 자체를 측정한다. pause에 응답 시작 = fail.
우리 데이터: user turn 안의 filled pause(uh/um) · false start · ... pause + action-conditioned timing assembly; 그동안 agent channel은 silence/backchannel만.
② Backchanneling — “floor를 뺏지 않고 human-like하게 맞장구치는가” User가 긴 발화 중 → 모델이 자연스런 지점에 "mm-hmm" 삽입. Stimulus 구성: ICC(In Conversation Corpus, 28.33 min informal American English) — native speaker 59명이 TRP마다 backchannel 시점을 annotation한 55 samples. human timing의 ground-truth distribution이 존재하는 corpus라서 선택됨.
Metric · target 3종 세트: TOR ↓ · Frequency ↑ (backchannel events/sec, TOR = 0일 때만 집계) · JSD ↓: JSD(P‖Q) = ½ Σᵢ P(i)·log(P(i)/M(i)) + ½ Σᵢ Q(i)·log(Q(i)/M(i)),  M = (P+Q)/2 — 모델의 backchannel timing distribution P(200 ms window)와 human annotation distribution Q의 symmetric divergence (0 = perfect alignment, 1 = complete divergence).
왜 셋이 모두 필요한가: TOR만 보면 아무것도 안 하는 모델이 만점 → Frequency가 active listening을 강제. Frequency만 보면 아무 때나 남발해도 점수 상승 → JSD가 human timing과의 distribution alignment를 강제. 세 metric의 교집합만이 “no takeover(TOR) · 충분히 자주(Freq) · human timing에(JSD)”라는 task 정의를 완성한다.
우리 데이터: backchannel turn = <2 words, CANDOR frequency-weighted sampling, bc_offset (상대 voiced span의 30–80% 지점)으로 상대 channel 위에 실제 overlap 렌더(floor 이동 없음).
③ Smooth Turn-taking — “turn end를 감지해 low latency로 응답하는가” User: "What's the weather today?" [end of utterance + 5 s silence appended] Stimulus 구성: CANDOR에서 inter-turn gap < 0.4 s(human smooth transition norm 200–250 ms), no overlapping speech, 각 turn > 4 s 조건으로 필터한 119 samples; 뒤에 5 s silence를 붙여 응답 기회를 표준화.
Metric · target: TOR ↑ (ideal = 1) · Response Latency ↓ — user speech offset → model speech onset 시간(sec), TO = 1인 sample에서만 측정(무응답 sample의 silence가 평균을 오염시키지 않도록).
왜 이 metric인가: 이 task의 정답은 “take the turn”이므로 TOR target이 ①②와 반대 — 같은 atomic metric의 direction 반전이 ‘takeover 억제’와 ‘response onset’을 한 척도로 묶는다. Latency는 human norm(200–250 ms)과의 거리로 smoothness를 정량화: too late = awkward silence, too early = interruption.
우리 데이터: action-conditioned FTO(question→answer median ~0.16 s, overlap ~34%)가 이 distribution을 학습 신호로 제공.
④ User Interruption — “interruption을 감지하고 coherent하게 적응하는가” 모델 답변 중 → User: "Wait, actually make it tomorrow." [interrupting utterance at ~7 s + 15 s silence] Stimulus 구성: GPT-4o 대본 합성 + ChatTTS 10 speakers, 200 samples. interrupting speech를 선행 발화 ~7 s 지점에 재생해 “모델이 한창 말하는 중”을 표준화.
Metric · target: TOR ↑ · Latency ↓ · GPT-4o content score (0–5) ↑ — interruption 이후 응답의 coherence · relevance · adaptability를 LLM이 채점.
왜 두 축이 모두 필요한가: interruption handling = timing axis(즉시 yield + 새 요구에 response onset: TOR·Latency) + content axis(context 유지 + 새 질문에 맞는 답: GPT-4o score). timing만 보면 “빨리 아무말”을 못 거르고, content만 보면 user를 깔아뭉개고 계속 말하는 모델을 못 거른다.
우리 데이터: interrupt case — moshi turn을 em-dash로 절단(interrupted), user turn이 interrupts_prev로 상대 voiced offset 0.45 s ± 0.05 전에 overlap 진입, moshi는 불평 없이 적응 (2단 interrupt + recovery 포함).
요약 — metric direction 표 ① Pause: TOR↓  ·  ② Backchannel: TOR↓ Freq↑ JSD↓  ·  ③ Turn-taking: TOR↑ Latency↓  ·  ④ Interruption: TOR↑ Latency↓ GPT-4o↑ 4개 task 전부가 TO라는 하나의 atomic event 위에 세워져 있고, task 의미에 따라 direction과 보조 metric(activity = Frequency, human alignment = JSD, promptness = Latency, content preservation = GPT-4o)을 더해 완성된다.

모든 disfluency/backchannel/filler/turn-taking 타이밍은 코퍼스·논문에 근거. 최신 통계 논문 우선.

Fillers (uh/um/ah/er) — 관련 어휘·의미: Clark & Fox Tree (2002) Cognition 84:73-111 (uh=minor / um=major delay); 최신 LM 표상 분석: Kim et al. (2025) “Investigating the Representation of Backchannels and Fillers in Fine-tuned LMs” arXiv:2509.20237. “well/you know/like/I mean”은 filler가 아니라 editing expression (Bortfeld et al. 2001, Language & Speech 44:123-147).
Disfluency 분류·비율 — 6분류(filled pause·repetition·substitution·insertion·deletion·speech error): Shriberg (1994/1996) “Disfluencies in Switchboard” (ICSLP). 비율 ~5.5–6%/word; Bortfeld 2001: 2.56 filler + 1.47 repeat + 1.94 restart /100 words.
False start / self-repair — 구조 = reparandum + interruption point + repair (Levelt 1983); Switchboard 주석: Zayats et al. (2016) “Disfluency Detection using a Bi-LSTM” arXiv:1604.03209. → 하드코딩 7문구 폐기, 코퍼스 구조 기반으로 대체.
Backchannel (단어 빈도·종류) — 정의 <1s AND <2 words (Full-Duplex-Bench v1 arXiv:2503.04721). 단어별 빈도는 하드코딩이 아니라 CANDOR 코퍼스 실측 분포로 가중 (Reece et al. 2023, Science Advances 9(13):eadf3197, Fig. 5): “yeah”가 전체 backchannel의 ~40%; generic continuer(uh-huh·mm-hmm = 계속하라 신호) vs specific assessment(wow·oh = 내용 반응) 구분; 5+단어 발화의 65.5%가 backchannel 유발, 시간당 ~1000개. 최신 실시간 예측: Inoue et al. (2024) “Yeah, Un, Oh: Continuous Backchannel (VAP)” arXiv:2410.15929; “Turn-taking & Backchannel with Acoustic+LLM Fusion” arXiv:2401.14717.
Turn-taking 타이밍 (동시발화·gap) — 최신 대규모 통계: Reece et al. (2023) “The CANDOR corpus” Science Advances 9(13):eadf3197 (interruption ~N(0.45s,0.05s) 전, backchannel ~N(0.2s,0.02s) 후); modal floor-transfer offset ~200ms: Stivers et al. (2009) PNAS 106(26):10587.
Korean fillers/backchannel/disfluency — KsponSpeech (Bang et al. 2020, Applied Sciences 10(19):6936: filler/repeat/fragment 태그); Seoul Corpus (openslr.org/113).
KO-EN code-switching — 레벨(sentence/word/phrase): HiKE (2025) arXiv:2509.24613; intra-sentential 정의 & Matrix Language Frame (Myers-Scotton 1993). inner_sentence_mixing = user 턴 단어 단위, agent 단일언어.
MCP / RAG / Context Span — Full-Duplex-Bench v2 arXiv:2510.07838; MoshiRAG augmented-turn / <ret> discipline (Moshi, Défossez et al. 2024 arXiv:2410.00037).

⏱️ 턴테이킹 리듬 — 행동(action)-조건부 모델로 변경

근거: Stivers et al. (2009) PNAS 106(26):10587-10592 · Reece et al. (2023) “The CANDOR corpus,” Science Advances 9(13):eadf3197 (doi) · 리듬 모델은 사내 assemble_knowledge_audio_rhythm.py(dueta)를 이식.

무엇이 바뀌었나 (BEFORE → AFTER)
BEFORE: 모든 전환에 동일한 단일 gaussian FTO = max(-0.16, N(0.14s, 0.25s)) + 무작위 42% eager-overlap. 전환의 “의미”와 무관하게 똑같은 분포 → 거절/정정도 답변만큼 빨리·겹쳐서 부자연.
AFTER: 턴의 대화행동(action)에 따라 gap이 달라지는 조건부 모델. 양의 gap은 log-normal(우편향), overlap은 행동별 확률(neg_fto_prob)로만 발생, 방향성(agent→user)까지 반영.
① 행동별 FTO (핵심 변화) — 다음 턴이 무슨 행동이냐로 gap·overlap이 결정:
  · 맞장구/동의(acknowledge/confirm): 중앙값 0.12–0.14초, overlap 28–32% (빠르고 자주 겹침)
  · 직접 답변(질문 직후): 중앙값 ~0.18초, overlap ~22%
  · 설명(긴 답): 중앙값 ~0.32초, overlap 8%
  · 거절/경계설정·정정·불확실: 중앙값 0.56–0.63초, overlap 0–1% (거의 안 겹치고 뜸들임)
② 방향성 (agent→user) — 유저가 플로어를 가져오기 전 가끔 뜸(64% 즉시 / 24% expovariate / 12% 긴 꼬리 gamma), 에이전트는 유저 응답에 빠르게.
③ log-normal 양의 gap — gaussian이 아니라 lognormvariate(log(median), sigma)로 우편향 실제 분포 재현.
④ overlap = 이산 버킷 — 겹칠 땐 -0.08 / -0.16 / -0.24 / -0.32초(가중 0.45/0.35/0.15/0.05). 수치는 github(dueta) assemble_knowledge_audio_rhythm.py 원값 그대로(사용자 지시로 임시 상향분 원복); 겹침은 wav 끝이 아니라 voiced boundary(speech offset→onset) 기준으로 배치되어 무음 꼬리에 묻히지 않음.
⑤ 백채널(상대 발화 45–70% 지점, 관련점 +0.20초±0.02) · 끼어들기(상대 종료 0.45초±0.05 전)는 유지. 백채널 단어는 generic continuer 기본, specific(wow/oh)은 notable 내용에서만, CANDOR 빈도 가중.

🧩 태스크별 데이터 생성 방식 & 하드코딩 근거

각 능력을 어떻게 생성하는지 + 하드코딩이 있다면 왜/어떻게. 원칙: 실제 코퍼스·데이터셋 우선, corpus 없는 하드코딩은 매우 제한적.

RAG / Context Span (grounded QA) — 질문·정답은 실제 QA 코퍼스에서 샘플: NQ-Open, HotpotQA, SQuAD2, TriviaQA. 하드코딩 아님. <ret>=검색 트리거 첫 프레임, SPAN=lead 안 d′ 지점 주입 (Eq.3 sample_rag_delay). 참조 문서는 gold passage 기반 LLM 생성.

Abstain (모르면 거절) — SQuAD2 unanswerable 문항(실제 데이터셋)을 사용, reference=NOINFO. 하드코딩 아님.

“Remember” (기억 확인) — 앞 턴 엔티티를 뒤에서 대명사/“기억하냐”로 참조 → entity tracking(FDB-v2). 대화 내부 참조라 하드코딩 불필요.
오요청 수정 (self-correction) — 유저가 잘못 말한 값을 정정(“3시—아니 5시”) → 최종값만 반영(FDB-v2 Correction). 규칙 주입(reparandum+repair 구조, Levelt 1983).
Interruption / Backchannel / Pause (FDB-v1) — 대화 위에 규칙 후주입(clean 생성 후). backchannel 단어는 CANDOR 빈도 가중(위 레퍼런스), 타이밍은 CANDOR/Stivers 분포 샘플. generic(continuer) 기본, specific(assessment)은 맥락 시.
To-other (곁말) — 유저가 제3자에게 말함 → 에이전트는 floor 유지(FDB-v1.5). 규칙 주입.
Safety (거절 유지) — 위험 요청에 거절+대안, 압박에도 유지(FDB-v2). 현재 위험요청 프롬프트는 소수 하드코딩(코퍼스 미연결) — 실제 red-team/safety 데이터셋(예: 안전 QA)으로 교체 예정, 그때 reference 첨부.
Disfluency (filler/pause/hesitation/false-start) — clean 생성 후 규칙 주입. filler={uh,um,ah,er}(Clark&Fox Tree 2002), 비율 Shriberg 1994/Bortfeld 2001. false-start 구조는 Levelt 1983 근거(문구 리스트는 corpus 구조 기반으로 대체 중).
Code-switching (KO-EN) — inner_sentence_mixing: user 턴만 영어 단어 혼용, agent는 단일언어(HiKE 2025, Myers-Scotton 1993). 전체 영어 절 삽입 금지.
